# AO Command v0.1.1 Operator Closeout

AO Command v0.1.1 advances the public, read-only operator command surface with
the operator readbacks and release qualification added since v0.1.0. It is not
a release publisher, production promoter, scheduler, or provider mutator.

## Release Notes

- Add bounded Mission, Atlas, Pulse, controlled-loop, and cross-repository
  operator readbacks while preserving read-only authority.
- Add strict operator-status validation, correlation identity, compatibility
  fixtures, and fail-closed evidence bounds.
- Add native Linux, macOS, and Windows candidate qualification.
- Add a manual release rehearsal that binds candidates to the exact source,
  version, tag, release-notes digest, and approved manifest before publication.
- Keep publication behind explicit live inputs, immutable plan verification,
  and the protected release environment.

## Ready Scope

- Inspect AO stack status and retained evidence supplied by an operator.
- Validate bounded evidence documents against their published contracts.
- Build and smoke-test native release candidates on their matching hosted
  operating systems.
- Assemble and verify an immutable release plan in dry-run mode.
- Report build identity through `ao-command version --json` without provider
  calls or repository mutation.

## Out Of Scope

- Publishing without explicit live release inputs and protected-environment
  approval.
- Promoting production, scheduling work, or mutating provider state.
- Accepting unbound, stale, malformed, or substituted release evidence.
- Autonomous cross-repository writes.

## Required Evidence Before Tagging

Run these checks from a clean worktree:

```sh
go test ./...
go vet ./...
go build -o bin/ao-command ./cmd/ao-command
scripts/public-readiness-audit.sh --forge ../ao-forge --out tmp/public-readiness-audit.json
scripts/release-preview-dry-run.sh --forge ../ao-forge --out tmp/release-preview --tag v0.1.1-preview
go run ./cmd/ao-command evidence --forge ../ao-forge --schema "$PWD/docs/contracts/release-preview-audit-v0.1.schema.json" --document "$PWD/tmp/release-preview/release-preview-audit.json"
scripts/install-verify-dry-run.sh --forge ../ao-forge --out tmp/install-verify
go run ./cmd/ao-command evidence --forge ../ao-forge --schema "$PWD/docs/contracts/install-verify-audit-v0.1.schema.json" --document "$PWD/tmp/install-verify/install-verify-audit.json"
scripts/release-governance-dry-run.sh --out tmp/release-governance --tag v0.1.1 --release-preview-audit tmp/release-preview/release-preview-audit.json --install-verify-audit tmp/install-verify/install-verify-audit.json
go run ./cmd/ao-command evidence --forge ../ao-forge --schema "$PWD/docs/contracts/release-governance-audit-v0.1.schema.json" --document "$PWD/tmp/release-governance/release-governance-audit.json"
```

The production-readiness audit must report `status=passed`,
`readiness_percent=100`, and `failed_checks=0`. The canonical release rehearsal
must run with `version=0.1.1`, `tag=v0.1.1`, the exact source commit, and
`dry_run=true` before any separate live-release decision.

## Publication State

This preparation does not create a tag or GitHub release. Publication remains a
separate operator decision governed by `.github/workflows/release-rehearsal.yml`
and its protected release environment.

## Safety Boundaries

- Operator commands remain read-only and provider-free.
- Dry runs do not create tags, releases, uploads, or deployments.
- The publisher requires exact source, manifest, candidate, notes, and plan
  bindings plus explicit protected-environment approval.
- Public-safe retained evidence remains governed by the public provenance manifest.
