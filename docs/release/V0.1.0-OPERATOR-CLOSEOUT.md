# AO Command v0.1.0 Operator Closeout

AO Command v0.1.0 is the public, read-only operator command surface for AO
Forge evidence. It is not a release publisher, production promoter, scheduler,
or provider mutator.

## Ready Scope

- Inspect AO Forge production-readiness status and next actions.
- Inspect AO Forge GoalRun state from an operator-provided GoalRun document.
- Validate evidence documents against AO Forge JSON Schema contracts.
- Rehearse AO Forge release-preview evidence from an explicit dry-run output
  directory.
- Produce AO Command release-preview, install-verify, and production-readiness
  dry-run evidence.
- Retain only public-safe dry-run evidence governed by
  `docs/operations/RETAINED-EVIDENCE.md` and
  `docs/operations/public-provenance-manifest.json`.

## Out Of Scope

- Publishing GitHub releases.
- Promoting production.
- Mutating branches, provider state, task queues, or control-plane records.
- Uploading CI artifacts by default.
- Running autonomous cross-repo work.

## Required Evidence Before Tagging

Run these checks from a clean worktree:

```sh
go test ./...
go vet ./...
go build -o bin/ao-command ./cmd/ao-command
scripts/public-readiness-audit.sh --forge ../ao-forge --out tmp/public-readiness-audit.json
scripts/release-preview-dry-run.sh --forge ../ao-forge --out tmp/release-preview --tag v0.1.0-preview
go run ./cmd/ao-command evidence --forge ../ao-forge --schema "$PWD/docs/contracts/release-preview-audit-v0.1.schema.json" --document "$PWD/tmp/release-preview/release-preview-audit.json"
scripts/install-verify-dry-run.sh --forge ../ao-forge --out tmp/install-verify
go run ./cmd/ao-command evidence --forge ../ao-forge --schema "$PWD/docs/contracts/install-verify-audit-v0.1.schema.json" --document "$PWD/tmp/install-verify/install-verify-audit.json"
scripts/production-readiness-audit.sh --repo uesugitorachiyo/ao-command --forge ../ao-forge --out tmp/production-readiness-audit.json
go run ./cmd/ao-command evidence --forge ../ao-forge --schema "$PWD/docs/contracts/production-readiness-audit-v0.1.schema.json" --document "$PWD/tmp/production-readiness-audit.json"
```

The production-readiness audit must report `status=passed`,
`readiness_percent=100`, and `failed_checks=0`.

## Publication State

The repository is public after passing the operator-approved public-readiness
audit. Do not add new public-visible evidence classes without updating the
public provenance manifest and rerunning the public-readiness audit.

## Remaining Operator Actions

- Merge the production-readiness PR.
- Require the `Production readiness audit` status check on `main`.
- Rerun the full admin-mode readiness audit after GitHub CLI authentication is
  fixed.
- Enable GitHub-verified signed commits after GitHub recognizes the signing key.
