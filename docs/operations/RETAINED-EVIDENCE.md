# Retained Evidence

AO Command may retain dry-run evidence only when it is public-safe and produced
by read-only verification paths. Retained evidence must never include provider
payloads, private control-plane exports, incident details, credentials, local
machine paths, or unpublished release artifacts.

## Public-Safe Evidence

The following dry-run evidence classes are public-safe when generated by the
checked-in scripts and validated against their contracts:

- release-preview audit JSON from `scripts/release-preview-dry-run.sh`;
- install verification audit JSON from `scripts/install-verify-dry-run.sh`;
- release governance audit JSON from `scripts/release-governance-dry-run.sh`;
- checksum manifests for generated dry-run outputs;
- command output from read-only `status`, `next`, `goals`, and `evidence`
  inspections when it references only public AO Forge fixtures.

The machine-readable retention policy is tracked in
`docs/operations/public-provenance-manifest.json`.

## Retention Rules

- Retain generated evidence under `tmp/` during local and CI dry runs.
- Do not upload CI artifacts by default.
- Do not publish retained evidence unless it is listed in the public provenance
  manifest and its contract validation passes.
- Re-run `scripts/public-readiness-audit.sh` before adding new evidence classes.
- Re-run `scripts/production-readiness-audit.sh` before using retained evidence
  as release support.

## Non-Retention Rules

Do not retain:

- secrets, credentials, private keys, tokens, or provider payloads;
- private operational logs, incidents, security reviews, or control-plane exports;
- local absolute paths outside generated dry-run output;
- release-publish, production-promotion, branch mutation, or provider mutation
  outputs.
